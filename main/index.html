Introduction to HTML

HTML (HyperText Markup Language) is the foundation of web development. It is used to structure content on the web by defining the elements of a web page. HTML doesn't define the appearance or behavior of elements—that's the role of CSS and JavaScript. HTML tells the browser what type of content to display and how it should be structured.

HTML, or HyperText Markup Language, forms the skeleton of any website. Think of HTML as the language through which we communicate with the web. While HTML tells the browser what to display, it doesn't make decisions on how to style or add interactivity to the content. That’s where CSS (Cascading Style Sheets) and JavaScript come in.

To start learning HTML, you need to understand the concept of "tags." Tags are special keywords that define the structure and content of a page. Each HTML element typically consists of an opening tag, some content, and a closing tag. For example, the text inside a paragraph is enclosed in the <p> tag. An HTML document consists of elements nested within each other.

Here's a basic example:

  <!DOCTYPE html>
  <html>
    <head>
      <title>My First Web Page</title>
    </head>
    <body>
      <h1>Welcome to My Website</h1>
      <p>This is my first web page.</p>
    </body>
  </html>

In this example:

  • <html>: Marks the beginning and end of an HTML document.
  • <head>: Contains meta-information about the document, such as the title that appears in the browser tab.
  • <body>: Contains all the content that is visible on the page, such as headings and paragraphs.
  • <h1>: Represents a top-level heading.
  • <p>: Represents a paragraph of text.

HTML is constantly evolving, and the latest version is HTML5, which introduced several new features, including multimedia support, improved forms, and semantic elements that make web content more accessible and meaningful.

HTML's role in web development is to provide structure. It gives developers a framework for organizing content in a logical, hierarchical manner. However, HTML by itself cannot control the appearance of elements. To style the page, you would use CSS (Cascading Style Sheets). CSS is responsible for setting colors, fonts, margins, and positioning of the content on a page. JavaScript, on the other hand, adds interactivity to a page, such as enabling form validation, animations, or even interactive games.

Basic HTML Structure

Every HTML document follows a basic structure that ensures a proper layout for the content. Understanding this structure is crucial for building organized and functional web pages. The structure of an HTML document typically contains a DOCTYPE declaration, the HTML root element, the head element, and the body element.

The basic HTML structure can be broken down into the following components:

  • <!DOCTYPE html>: This declaration tells the browser that the document is written in HTML5. It's important because it ensures that the page is rendered in the correct mode by modern browsers.
  • <html>: The root element of the page. Everything inside this tag is considered part of the HTML document.
  • <head>: Contains metadata, links to external files (like CSS and JavaScript), and other information that doesn't directly appear in the visible content of the page.
  • <body>: This is where all the visible content of the page goes—texts, images, links, forms, etc.

The head section typically includes:

  • <meta>: Used to define metadata about the HTML document, such as character encoding or page description for search engines.
  • <title>: Defines the title of the webpage, which appears in the browser’s tab.
  • <link>: Links external stylesheets to the document.
  • <script>: Links or includes JavaScript code that enhances the interactivity of the webpage.

Here’s an example of a basic HTML structure:

  <!DOCTYPE html>
  <html lang="en">
    <head>
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <title>My First Web Page</title>
    </head>
    <body>
      <h1>Hello, World!</h1>
      <p>This is my first website built using HTML.</p>
    </body>
  </html>

In this structure:

  • <meta charset="UTF-8">: Specifies the character encoding used for the document.
  • <meta name="viewport">: Defines how the page should be displayed on different screen sizes (important for mobile responsiveness).
  • <h1>: This is the main heading of the page, and it's typically the most important part of your content.
  • <p>: Paragraph text. Here, we are simply introducing the web page.

It's essential to understand the significance of the <head> and <body> sections. The <head> contains everything that is used to control the document's settings, such as metadata, links to CSS files, and scripts. The <body> is where all your actual content that users interact with will be placed.

With this structure in mind, you can now start building more complex web pages by adding additional elements and more content inside the <body> section. Keep in mind that HTML forms the skeleton of your webpage, and how you organize your HTML structure determines how well your webpage functions and how easily it can be styled and made interactive with CSS and JavaScript.

HTML Elements

HTML elements are the basic building blocks of HTML. Each element is typically enclosed in opening and closing tags, with some content in between. These elements can also have attributes, which provide additional information about the element's behavior or styling.

HTML elements are essential for marking up the content of a webpage. An element consists of the opening tag, content, and closing tag. For example:

  <p>This is a paragraph of text.</p>

In this case:

  • <p>: Represents the paragraph element.
  • </p>: This is the closing tag for the paragraph.

Each HTML element is used to define specific parts of the webpage. For example:

  • <h1> to <h6>: These tags represent different levels of headings, from the most important (<h1>) to the least important (<h6>).
  • <a>: The anchor element, used for creating hyperlinks.
  • <img>: The image element, used for embedding images into the page.
  • <ul>: Unordered list element, used to create lists with bullet points.
  • <ol>: Ordered list element, used to create numbered lists.
  • <li>: List item element, used inside both ordered and unordered lists.

HTML elements can also include attributes. Attributes provide additional information about an element. For example, the <a> element has an href attribute that specifies the URL the link should navigate to:

  <a href="https://www.example.com">Visit Example</a>

Attributes are always written within the opening tag, and they are typically in the form of a name-value pair (like href="https://www.example.com"). Not all elements have attributes, but many do, and they are essential for adding functionality or style to your elements.

Understanding the role of elements and attributes will allow you to create structured, meaningful HTML documents. Every piece of content on the page, from text and links to images and forms, is defined using HTML elements.

HTML Syntax

HTML tags are written using angle brackets. Some elements are self-closing. Attributes provide extra information.

  <img src="image.jpg" alt="A sample image" />
  <a href="https://example.com">Visit</a>

Headings

HTML provides 6 levels of headings, from <h1> (most important) to <h6> (least important).

  <h1>Main Title</h1>
  <h2>Subtitle</h2>

Links

Links are created using the <a> tag with the href attribute to point to the destination.

  <a href="https://example.com">Click Here</a>

Images

Use the <img> tag to display images. Always include the alt attribute for accessibility.

  <img src="cat.jpg" alt="A cute cat">

Lists

HTML supports ordered lists (<ol>) and unordered lists (<ul>). Each item is wrapped in an <li> tag.

  <ul>
    <li>Apple</li>
    <li>Banana</li>
  </ul>
  
  <ol>
    <li>First</li>
    <li>Second</li>
  </ol>

Tables

Tables display data in rows and columns using <table>, <tr>, <th>, and <td> tags.

  <table>
    <tr><th>Name</th><th>Age</th></tr>
    <tr><td>Alice</td><td>30</td></tr>
  </table>

Semantic Elements

Semantic HTML uses tags that describe their meaning, making code more readable and accessible.

  <header></header>
  <nav></nav>
  <article></article>
  <footer></footer>

Semantic Meaning

Semantic elements give meaning to web content. They help search engines and screen readers understand the structure.

  <section>Main section content</section>
  <aside>Side info like ads or links</aside>

Forms

HTML forms collect user input using fields like text, checkboxes, radios, and more.

  <form action="/submit" method="post">
    <input type="text" name="username">
    <input type="submit" value="Send">
  </form>

Form Validation

HTML provides built-in validation like required fields, patterns, and input types.

  <input type="email" required>
  <input type="text" pattern="[A-Za-z]{3,}" title="Min 3 letters">

Audio and Video

Use the <audio> and <video> tags to embed media. Include controls for playback.

  <audio controls>
    <source src="music.mp3" type="audio/mpeg">
  </audio>
  
  <video controls width="320">
    <source src="movie.mp4" type="video/mp4">
  </video>

Canvas

<canvas> is used for rendering graphics with JavaScript — like games or drawings.

  <canvas id="myCanvas" width="200" height="100"></canvas>
  
  <script>
    const canvas = document.getElementById('myCanvas');
    const ctx = canvas.getContext('2d');
    ctx.fillStyle = 'blue';
    ctx.fillRect(10, 10, 150, 75);
  </script>

SVG (Scalable Vector Graphics)

SVG defines graphics using XML. It's ideal for resolution-independent images like logos.

  <svg width="100" height="100">
    <circle cx="50" cy="50" r="40" stroke="black" stroke-width="3" fill="red" />
  </svg>

Microformats

Microformats use HTML classes and attributes to describe metadata, like people or events, in a machine-readable way.

  <div class="h-card">
    <span class="p-name">Jane Doe</span>
    <a class="u-url" href="http://janedoe.com">Website</a>
  </div>

Metadata

Metadata provides info about the document — used by browsers, search engines, and social media.

  <meta charset="UTF-8">
  <meta name="description" content="Learn HTML for free">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

Accessibility Basics

Accessible HTML helps users with disabilities navigate content using screen readers or keyboards.

  <img src="cat.jpg" alt="A fluffy grey cat lounging" />
  <button aria-label="Close">X</button>

ARIA Attributes

ARIA (Accessible Rich Internet Applications) attributes describe elements for assistive tech.

  <div role="navigation" aria-label="Main menu">
    <button aria-expanded="false">Toggle Menu</button>
  </div>

HTML Best Practices

Write clean, semantic, and accessible HTML. Use proper indentation, alt attributes, and meaningful tags.

  • Use semantic elements
  • Include alt text for images
  • Validate your HTML

HTML Tools

Use tools to test and improve your HTML:

  • W3C Validator
  • CodePen
  • Google web.dev

Web Components

Web components are reusable custom elements using Shadow DOM and JavaScript.

  <template id="my-component">
    <style> p { color: blue; } </style>
    <p>Hello Web Component!</p>
  </template>

HTML5 APIs

HTML5 introduced APIs for geolocation, drag & drop, offline storage, and more.

  <script>
  navigator.geolocation.getCurrentPosition(function(pos) {
    console.log(pos.coords.latitude, pos.coords.longitude);
  });
  </script>
